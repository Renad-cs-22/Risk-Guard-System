🛡️ RiskGuard System (Maker-Checker & Immutable Audit Log)

نظام محاكاة أمني متكامل ومتقدم مبني بلغة **Python**، مصمم خصيصاً للشركات والمؤسسات المالية (FinTech) لإدارة مخاطر العمليات وضمان الامتثال الرقابي. يعتمد النظام على تطبيق ضوابط صارمة لحماية البيانات ومنع الاحتيال الداخلي.

---

## 🔥 الميزات الرئيسية (Key Features)

* **الفصل الصارم بين الواجبات (Segregation of Duties - SoD)**: يمنع النظام برمجياً ومنطقياً أي موظف أنشأ عملية (Maker) من مراجعتها أو اعتمادها بنفسه (Checker).
* **سجل تدقيق غير قابل للتعديل (Immutable Audit Trail)**: مستوحى من مفهوم سلسلة الكتل (Blockchain)، حيث يتم ربط كل سجل بالذي قبله عبر تشفير `SHA-256`؛ مما يجعل أي محاولة لتعديل السجلات القديمة مكشوفة فوراً.
* **تقييم تلقائي للمخاطر (Automated Risk Assessment)**: تصنيف فوري لمستوى خطورة العمليات (عالي / متوسط) بناءً على القيم المالية المدخلة.
* **معرفات فريدة ومقاومة للتكرار**: توليد معرفات عمليات حصرية تستخدم مزيجاً من التاريخ اليومي والرموز العشوائية المشفرة عبر `UUID4`.
* **دعم كامل للغة العربية**: معالجة النصوص والترميز بأسلوب يضمن ثبات قيمة الـ Hash التشفيري دون تضارب.

---

## 🛠️ التكنولوجيا المستخدمة (Tech Stack)

* **اللغة الأساسية**: Python 3.x
* **المكتبات المدمجة**: `hashlib` (للتشفير)، `json` (لهيكلة البيانات)، `uuid` (لتوليد المعرفات)، `datetime` (للتوقيت الزمني).

## 🚀 طريقة التشغيل والتجربة (Quick Start)

1. قم بتحميل أو استنساخ المستودع (Clone):
   ```bash
   git clone https://github.com
   ```
2. انتقل إلى مجلد المشروع:
   ```bash
   cd RiskGuard-System
   ```
3. قم بتشغيل النظام مباشرة:
   ```bash
   python main.py
   ```

## 📊 سيناريو المحاكاة داخل الكود (Simulation Scenario)
عند تشغيل الكود، يقوم النظام تلقائياً بتجربة:
1. إنشاء عمليات مالية بمبالغ مختلفة وتقييم مخاطرها تلقائياً.
2. محاولة اختراق السياسات الأمنية (موظف يحاول اعتماد عمليته بنفسه) وإظهار رسالة المنع الأمنية `🚨 [انتهاك أمني]`.
3. تمرير العمليات واعتمادها بشكل صحيح عبر مدققين مختلفين.
4. طباعة سجل التدقيق النهائي متضمناً الـ Hash الحالي والـ Hash السابق لكل كتلة بيانات.
